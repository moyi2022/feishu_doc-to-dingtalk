# 飞书文档迁移到钉钉文档 - 技术报告

## 1. 项目概述

### 1.1 项目目标

开发一个本地工具，支持将飞书文档一键迁移到钉钉知识库，保持文档格式完整性。

### 1.2 功能特点

- 支持单个文档迁移
- 支持批量文档迁移（整个知识库）
- 保持文档格式（文本、标题、列表、代码块、引用等）
- Web 界面 + 命令行两种使用方式
- 本地执行，数据安全

### 1.3 技术栈

| 组件 | 技术 |
|------|------|
| 后端 | Python Flask |
| 前端 | HTML + JavaScript + CSS |
| 飞书 API | REST API |
| 钉钉 API | REST API + 阿里云 SDK |
| 格式转换 | Markdown |

## 3. 核心模块设计

### 3.1 飞书文档导出 (feishu_exporter.py)

**功能：** 调用飞书开放平台 API，导出文档为 Markdown 格式。

**API 调用流程：**

```
1. 获取 tenant_access_token
   POST /auth/v3/tenant_access_token/internal

2. 获取文档元数据（包含标题）
   GET /docx/v1/documents/{document_id}

3. 获取文档块内容
   GET /docx/v1/documents/{document_id}/blocks

4. 解析块类型，转换为 Markdown
```

**支持的块类型映射：**

| 飞书块类型 | 类型码 | Markdown 格式 |
|-----------|--------|--------------|
| 文本段落 | 2 | 普通文本 |
| 标题1-9 | 3-11 | # ~ ######### |
| 无序列表 | 12 | - item |
| 有序列表 | 13 | 1. item |
| 代码块 | 14 | ```code``` |
| 引用 | 15 | > quote |
| 待办事项 | 17 | - [ ] todo |
| 分割线 | 22 | --- |

### 3.2 钉钉文档导入 (dingtalk_importer.py)

**功能：** 调用钉钉开放平台 API，创建知识库文档并写入内容。

**API 调用流程：**

```
1. 获取 access_token
   POST /v1.0/oauth2/{corp_id}/token

2. 创建文档
   POST /v1.0/doc/workspaces/{workspace_id}/docs
   Body: { name, docType, operatorId }

3. 写入文档内容
   POST /v1.0/doc/suites/documents/{doc_key}/overwriteContent
   Query: operatorId
   Body: { content }
```

### 3.3 迁移服务协调 (migration_service.py)

**功能：** 整合飞书导出和钉钉导入，处理单个/批量迁移逻辑。

**迁移流程：**

```
飞书文档 URL → 提取文档 ID → 导出 Markdown → 创建钉钉文档 → 写入内容
```

## 4. 开发过程中解决的问题

### 4.1 钉钉内容更新 API 404 错误

**问题描述：**
调用钉钉文档内容更新接口时，返回 HTTP 404 错误。

**排查过程：**
1. 检查官方 SDK 源码，找到正确端点
2. 发现 SDK 内部使用的是 `/v1.0/doc/suites/documents/{doc_key}/overwriteContent`

**解决方案：**
- 正确端点：`POST /v1.0/doc/suites/documents/{doc_key}/overwriteContent`
- 错误端点：`PUT /v1.0/doc/docs/{doc_key}/content`

```python
# 修正后的代码
url = f"{self.base_url}/v1.0/doc/suites/documents/{doc_key}/overwriteContent"
response = requests.post(url, json=payload, headers=headers, params=params)
```

### 4.2 dataType 参数导致的 400 错误

**问题描述：**
内容更新 API 返回 "Unsupported dataType" 错误。

**排查过程：**
1. 分析 API 响应，发现 `dataType` 字段不被支持
2. 测试发现钉钉 API 自动识别内容格式

**解决方案：**
移除 `dataType` 字段，只传递 `content` 字段：

```python
# 错误写法
payload = {
    "content": content,
    "dataType": "html"  # 不支持
}

# 正确写法
payload = {
    "content": content  # API 自动识别格式
}
```

### 4.3 operatorId 参数位置问题

**问题描述：**
创建文档和更新内容时，operatorId 的传递位置不同。

**解决方案：**
- 创建文档：operatorId 在请求体 (Body)
- 更新内容：operatorId 在查询参数 (Query)

```python
# 创建文档 - operatorId 在 body
payload = {
    "name": title,
    "docType": "DOC",
    "operatorId": user_id  # 在 body 中
}

# 更新内容 - operatorId 在 query params
params = {"operatorId": user_id}  # 作为查询参数
response = requests.post(url, json=payload, params=params)
```

### 4.4 unionId vs userId 问题

**问题描述：**
钉钉文档 API 需要使用 unionId，而不是 userId，否则返回 500 错误。

**解决方案：**
1. 开发 unionId 自动获取工具 (`get_unionid.py`)
2. 添加 Web 界面一键获取功能
3. 支持通过 userId 或手机号查询

**获取 unionId 的 API 调用：**

```python
# 方法1: 通过 userId 获取
url = f"https://oapi.dingtalk.com/topapi/v2/user/get?access_token={token}"
response = requests.post(url, json={'userid': user_id})
unionid = response.json()['result']['unionid']

# 方法2: 先通过手机号获取 userId，再获取 unionId
url = f"https://oapi.dingtalk.com/topapi/v2/user/getbymobile?access_token={token}"
response = requests.post(url, json={'mobile': mobile})
user_id = response.json()['result']['userid']
# 然后调用方法1
```

**所需权限：**
- `Contact.User.Read` - 通讯录只读
- `qyapi_get_member_by_mobile` - 通过手机号获取用户
- `qyapi_get_member` - 成员信息读取

### 4.5 文档标题问题

**问题描述：**
迁移后的钉钉文档标题显示为 "迁移文档：{文档ID}"，而不是飞书文档的原始标题。

**解决方案：**
修改 `export_document` 方法返回字典，同时返回标题和内容：

```python
# feishu_exporter.py
def export_document(self, document_id: str) -> Optional[Dict[str, str]]:
    # ... 获取文档元数据
    title = document.get("title", "未命名文档")

    return {
        'title': title,
        'content': markdown
    }

# migration_service.py
export_result = self.feishu.export_document(document_id)
doc_title = export_result.get('title', '未命名文档')
doc_url = self.dingtalk.create_document(workspace_id, doc_title, markdown_content)
```

### 4.6 内容格式选择

**问题描述：**
需要确定钉钉文档使用 HTML 还是 Markdown 格式。

**测试结果：**
| 格式 | HTTP 响应 | 渲染效果 |
|------|----------|---------|
| HTML | 200 | 正确渲染 |
| Markdown | 200 | 正确渲染 |
| 纯文本 | 200 | 纯文本显示 |

**最终方案：**
选择 Markdown 格式，原因：
1. 飞书导出已是 Markdown，无需额外转换
2. 用户后续编辑更方便
3. 文件更轻量，可读性强

## 5. API 凭证配置

### 5.1 飞书凭证

**获取路径：** 飞书开放平台 → 我的应用 → 凭证与基础信息

**所需权限：**
- `docx:document:read` - 查看新版文档
- `wiki:wiki:read` - 查看知识库
- `drive:drive:readonly` - 下载云文档中的图片和附件

### 5.2 钉钉凭证

**获取路径：** 钉钉开放平台 → 应用开发 → 企业内部开发

**所需权限：**
- 知识库文档写权限
- 企业存储文件写权限
- 通讯录只读权限（用于获取 unionId）

## 6. 使用说明

### 6.1 安装依赖

```bash
pip install -r requirements.txt
```

### 6.2 启动 Web 界面

```bash
python app.py
```

访问 http://localhost:5000

### 6.3 命令行使用

编辑 `run_migration.py`，填写凭证：

```python
feishu_creds = {
    'app_id': '你的飞书AppID',
    'app_secret': '你的飞书AppSecret'
}

dingtalk_creds = {
    'client_id': '你的钉钉ClientID',
    'client_secret': '你的钉钉ClientSecret',
    'corp_id': '你的企业CorpId',
    'user_id': '操作用户unionId'
}
```

运行：
```bash
python run_migration.py
```

## 7. 技术总结

### 7.1 关键技术点

1. **API 端点选择**：钉钉 SDK 源码是查找正确 API 端点的可靠来源
2. **参数位置**：不同 API 对同一参数（如 operatorId）的位置要求可能不同
3. **权限管理**：unionId 获取需要额外的通讯录权限
4. **格式兼容**：Markdown 格式在钉钉文档中渲染良好

### 7.2 踩坑记录

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 404 错误 | API 端点错误 | 查阅 SDK 源码确定正确端点 |
| 400 dataType | 不支持的参数 | 移除 dataType 字段 |
| 500 错误 | 使用 userId 而非 unionId | 获取正确的 unionId |
| 标题错误 | 未获取文档标题 | 返回标题+内容的字典结构 |

### 7.3 后续优化方向

1. **图片迁移**：支持文档中图片的自动迁移
2. **增量同步**：支持文档变更的增量更新
3. **错误重试**：添加 API 调用失败的重试机制
4. **进度推送**：WebSocket 实时推送迁移进度
